Codex must read this before implementation.

`cl_platform` is a demo prototype of a state-level platform for regulating cultural and entertainment events in Belarus.

Audience: ministries and institutional stakeholders. Goal: convincing demo flow, not production backend.

Core chain that must not be broken:

Candidate organizer -> Organizer -> Event application -> Control Center -> Approved event -> Sellers / retail / ticket flow.

DTCM/Dubai is only a product reference. Use its principles: multi-stage application, save/resume, documents, checks, fees, controlled sellers.
