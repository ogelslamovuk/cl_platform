## Цель итерации

Сделать удобный инструмент расценивания мест в зрительном зале:

Заявка -> Билеты и тарифы -> текущая схема зала -> массовое назначение тарифа -> точечные исключения -> один демонстрационный льготный тариф -> сохранение -> read-only summary в Центре Управления.

Важно:

- показы/sessions/performances в этой итерации не делать;
- новый конструктор не создавать;
- если текущая схема проще, использовать multi-seat selection на существующих местах.
