# Acceptance checklist

Результат принят только если выполнены все пункты.

## `/main` и `/proto`

- [ ] `/main` не содержит popup / блок «Инструменты прототипа».
- [ ] `/main` не содержит кнопку «Запустить демо-сценарий».
- [ ] `/proto` не содержит кнопку «Запустить демо-сценарий».
- [ ] Быстрые входы `/proto` не сломаны.

## Единая заявка мероприятия

- [ ] В кабинете организатора клик по строке заявки открывает 9-step compliance view.
- [ ] Кнопка «Просмотр» заявки ведёт туда же или к тому же detail-view.
- [ ] В Центре Управления «Заявки мероприятий» отображаются списком / таблицей.
- [ ] Клик по заявке в Центре Управления открывает 9-step compliance view в admin-mode.
- [ ] Не создан второй независимый интерфейс заявки.
- [ ] Admin-mode содержит комментарий и действия approve / return / reject.

## Документы

- [ ] Документы заявки имеют понятные названия.
- [ ] Технические file names не являются основным названием.
- [ ] Документы открываются визуальным preview.
- [ ] Mock-паспорта выглядят как ретро-бюрократические документы.
- [ ] В mock-паспортах есть визуальные портреты.
- [ ] Данные документов вымышленные и маркированы `DEMO`.

## Тарифы

- [ ] Предзаполненная раскраска тарифов не идёт вертикальными колонками.
- [ ] Тарифы распределены горизонтальными зонами по удалению от сцены.
- [ ] Нет шахматной/вертикально-полосатой раскраски тарифов.

## Региональные режимы

- [ ] Есть режимы: республика, Могилёвская область, Гродненская область.
- [ ] В Могилёвской области есть demo-заявки/мероприятия/площадки.
- [ ] В Гродненской области есть demo-заявки/мероприятия/площадки.
- [ ] При переключении региона пересчитываются все KPI.
- [ ] Заявки мероприятий фильтруются по региону.
- [ ] Региональный режим не показывает чужие области.

## Dashboard Центра Управления

- [ ] KPI «Нарушения» удалён или заменён.
- [ ] KPI «Подозрительные» удалён или заменён.
- [ ] Есть KPI «К выдаче удостоверений».
- [ ] Есть KPI «Ожидают публикации».
- [ ] Новые KPI ведут в разные логичные разделы/фильтры.
- [ ] Новые KPI учитывают региональный режим.

## SeatMap templates

- [ ] Есть 6 шаблонов: театр, амфитеатр, цирк, малая арена, ледовая арена, дворец спорта.
- [ ] Театр и амфитеатр визуально разные.
- [ ] Цирк читается как круглая/кольцевая схема.
- [ ] Малая арена не копия цирка.
- [ ] Ледовая арена не копия дворца спорта.
- [ ] Нет наложений блоков на арену/сцену/другие блоки.
- [ ] Open-air не добавлен как seatmap template.
- [ ] В блоке «Шаблоны посадки» есть пиктограммы.
- [ ] Нет длинных описаний на карточках шаблонов.

## B2C

- [ ] Постеры вертикальные.
- [ ] Постеры визуально разные, не один шаблон разного цвета.
- [ ] Постеры похожи на event posters.
- [ ] Нет повторяющихся служебных подписей «Белорусская культурная программа» / «афиша мероприятия».
- [ ] Нет дублей событий с одинаковым названием и постером.
- [ ] События со схемой открывают выбор мест.
- [ ] События без схемы / open-air не открывают seatmap.
- [ ] Seatmap modal открывается отцентровано и читаемо.

## Demo-data

- [ ] Нет мероприятий на 3000–3500 мест.
- [ ] Open-air максимум около 1500 мест.
- [ ] Open-air без схемы.
- [ ] События со схемой не больше 500 мест.
- [ ] `kvitki.by` нигде не осталось.
- [ ] Используется `bezkassira` / «Безкассира».

## Build / delivery

- [ ] `npm run build` проходит.
- [ ] Smoke-check проходит на `/main`, `/proto`, `/organizer`, `/admin`, `/channel`, `/demo`.
- [ ] PR создан и merged.
- [ ] GitHub Pages deploy successful.
- [ ] Live routes проверены.
