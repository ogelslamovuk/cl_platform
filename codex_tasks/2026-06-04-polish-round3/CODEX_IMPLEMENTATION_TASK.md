## 5. Детализация реализации

### 5.1. `/main` и `/proto`

- На `/main` убрать блок / popup «Инструменты прототипа».
- На `/main` и `/proto` убрать кнопку «Запустить демо-сценарий».
- Быстрые входы `/proto`, которые уже открывают новые вкладки, не переделывать.

### 5.2. Организатор → Заявки

- Список заявок оставить.
- Сделать строку заявки кликабельной.
- Клик должен открыть 9-step compliance view этой заявки.
- Кнопка «Просмотр» может вызывать тот же переход.

### 5.3. Центр Управления → Заявки мероприятий

- Заменить большую карточку-простыню на список / таблицу заявок.
- Клик по заявке открывает detail-view через тот же 9-step compliance UI.
- В admin-mode показывать админские действия и комментарий.
- Не создавать второй UI.

### 5.4. Документы

- Документам дать понятные названия.
- Техническое имя файла показывать мелко вторым уровнем.
- Preview сделать визуальным.

### 5.5. Mock-паспорта

- Усилить ретро-бюрократическую пиксельную стилистику.

### 5.6. Тарифы

- Исправить предзаполненную раскраску тарифов.
- Не использовать вертикальные колонки.
- Сделать горизонтальные зоны по удалению от сцены.

### 5.7. Региональные режимы

- Проверить и исправить фильтрацию всех регионально-зависимых KPI.
- Добавить/поправить demo-data для Могилёвской и Гродненской областей.

### 5.9. SeatMap templates

- Переделать 6 шаблонов так, чтобы они были визуально разными.
