# Polish round 3: заявка как единая сущность, SeatMap-шаблоны, B2C-постеры, регионы

## Цель

Довести текущий demo-flow `cl_platform` до более цельного demo-ready состояния.

Это polish-итерация. Не делать production backend и не ломать текущий flow.

Главная цепочка, которую нельзя ломать:

`Кандидат в организаторы -> Организатор -> Заявка на мероприятие -> Центр Управления -> Одобренное событие -> Реселлеры / розница / билеты`

## Основные блоки

1. Убрать лишние элементы с `/main` и `/proto`.
2. Сделать заявку мероприятия единой сущностью с одним 9-step compliance UI.
3. В кабинете организатора открывать заявку через этот 9-step UI.
4. В Центре Управления показывать заявки мероприятий списком, detail-view открывать через тот же 9-step UI в admin-mode.
5. Подписать документы человеческими названиями и показывать визуальный preview.
6. Усилить mock-паспорта участников в ретро-бюрократическом стиле.
7. Исправить тарифы на схеме: горизонтальные зоны вместо вертикальных полос.
8. Исправить региональные demo-data и KPI для Могилёвской и Гродненской областей.
9. Заменить KPI «Нарушения» и «Подозрительные».
10. Довести SeatMap templates до demo-ready качества.
11. Заменить текстовые карточки шаблонов на пиктограммы.
12. Усилить B2C-постеры и убрать дубли.
13. Исправить стартовое положение seatmap modal в B2C.
14. Не перерабатывать глубоко раздел «Контроль».
