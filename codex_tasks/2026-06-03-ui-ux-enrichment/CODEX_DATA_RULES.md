# Demo-data and product data rules

## 1. Regions

Область — основной уровень региональных прав.

Использовать человекочитаемые значения:

- `Минск`
- `Минская область`
- `Могилёвская область`
- `Гомельская область`
- `Гродненская область`
- `Брестская область`
- `Витебская область`

Город хранить и показывать отдельно.

Регион заявки определяется по месту проведения / площадке.

## 2. Demo admin accounts

Нужно иметь два demo-режима / аккаунта Центра Управления:

1. `Республиканский уровень` / `Супер-админ`
   - видит всю страну;
   - фильтр области может показывать `Все области`.

2. `Могилёвская область` / `Региональный сотрудник`
   - видит только Могилёвскую область;
   - не видит заявки, площадки, мероприятия и контроль других областей.

Не делать районный уровень.

## 3. Participants and documents

Документы участников — mock only.

Разрешённые mock-названия файлов:

- `passport_artist_01_mock.pdf`
- `passport_artist_02_mock.pdf`
- `passport_tech_01_mock.pdf`
- `group_roster_mock.pdf`
- `venue_contract_mock.pdf`

Нельзя использовать настоящие номера паспортов.

Нельзя использовать реальные персональные данные.

ФИО использовать вымышленные.

## 4. Fees

Не добавлять неподтверждённые сборы.

Пошлина рассчитывается только через вместимость / количество билетов и БВ.

Состояние освобождения от пошлины должно иметь причину.

Допустимые demo-причины:

- `Только белорусские исполнители`;
- `Государственная организация культуры`;
- `Льготный demo-сценарий`.

## 5. Seatmap vs capacity-only

События с seatmap:

- имеют места;
- вместимость не больше 500;
- в B2C используют выбор места.

Capacity-only / open-air события:

- могут быть больше 500;
- не имеют seatmap;
- в B2C используют прямую покупку входного билета;
- в реестре показывают `Без схемы`.

## 6. Control events

Разрешённые demo-кейсы контроля:

- повторное погашение одного билета;
- билет погашен не на той площадке;
- превышение лимита выпущенных билетов относительно утверждённой вместимости;
- продажа оператором, у которого нет доступа к событию;
- расхождение количества выпущенных билетов и утверждённой квоты;
- подозрительная серия возвратов;
- билет с невалидным QR/barcode.

Запрещённые demo-кейсы контроля:

- попытка публикации без оплаты пошлины;
- продажа отказана из-за отсутствия билетов;
- техническая ошибка без бизнесового объекта.

## 7. Operations

Журнал операций показывает действия системы:

- продажа успешна;
- возврат выполнен;
- билет погашен;
- событие опубликовано;
- пошлина оплачена;
- лимит билетов обновлён;
- оператор подключён.

Билеты и операции не объединять.

## 8. Posters

Постеры вертикальные.

Пропорция: 2:3 или 3:4.

Минимум 2–3 разных poster-layout шаблона.

Запрещён текст на постере:

`Центр управления · Demo`.

Постеры должны быть разными, но в едином стиле.
