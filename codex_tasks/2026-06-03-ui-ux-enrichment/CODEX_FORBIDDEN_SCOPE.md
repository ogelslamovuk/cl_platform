# CODEX_FORBIDDEN_SCOPE.md

# Forbidden scope

Эти ограничения обязательны для всей задачи.

## 1. Flow

Нельзя ломать основной flow:

`Кандидат в организаторы → Организатор → Заявка на мероприятие → Центр Управления → Одобренное событие → Операторы / розница / билеты`.

Нельзя удалять или отключать существующие рабочие сценарии.

## 2. Architecture

Нельзя:

- делать broad refactor;
- переписывать приложение с нуля;
- менять роутинг без прямой необходимости для expected result;
- создавать второй независимый store;
- создавать второй независимый реестр площадок;
- создавать второй независимый seatmap engine;
- создавать второй независимый B2C purchase flow;
- дублировать существующие сущности.

## 3. Backend / integrations

Нельзя добавлять:

- production backend;
- реальные API МВД;
- реальные API миграции;
- реальные API Минкульта / исполкома;
- реальный платёжный шлюз;
- реальные загрузки документов на сервер;
- реальные паспортные данные;
- реальные персональные данные.

Все проверки, документы и платежи — demo/mock UI.

## 4. DTCM / Dubai

DTCM используется только как продуктовый референс.

Нельзя создавать user-facing термины или сущности:

- DTCM;
- Dubai;
- NEN;
- eForm;
- ePermit;
- New Event Notification;
- TourismID;
- Manager’s Cheque.

Беларуский прототип должен использовать текущие сущности и русский UI.

## 5. SeatMap

Нельзя:

- добавлять open-air как seatmap template;
- рисовать тысячи мест;
- ломать покупку для событий без схемы;
- удалять текущий working SeatMap без fallback;
- делать второй конструктор;
- создавать второй реестр площадок.

Open-air / capacity-only события существуют без схемы.

## 6. UI text

Нельзя показывать raw technical labels:

- `sell`;
- `redeem`;
- `refund`;
- `issue`;
- `published`;
- `approved`;
- `issued`.

Использовать человекочитаемые русские статусы.

Все новые видимые UI-тексты — только русский язык.

## 7. Demo-data

Нельзя оставлять:

- невозможные операции отказа продажи из-за отсутствия билетов;
- крупные события с обычной seatmap-схемой;
- одинаковые poster-клоны;
- постеры с `Центр управления · Demo`;
- случайные mock-сущности без связи со сквозным flow.

## 8. Code quality

Нельзя:

- дублировать JSX-блоки;
- дублировать массивы;
- оставлять старый и новый элемент рядом;
- закрывать проблему декоративной заглушкой;
- сдавать без build;
- сдавать без smoke-check;
- сдавать без live-check.
