# CODEX_EXPECTED_RESULT

## Главный ожидаемый результат

После итерации прототип должен выглядеть как цельная, аккуратно отполированная demo-система.

Нужно устранить логические противоречия после предыдущего большого прохода:

- данные должны быть сквозными;
- постеры должны быть сквозными;
- документы должны быть визуальными;
- региональная модель должна быть наполнена данными;
- B2C должен вести себя как настоящая витрина;
- open-air не должен притворяться залом со схемой;
- seatmap templates должны реально впечатлять и отличаться;
- compliance tooltips должны быть содержательными;
- `/proto` должен быть удобным инструментом демонстрации.

Это не задача на перестройку архитектуры. Это polish / consistency pass.

## 1. Mock-data

Итоговое состояние:

- нет событий на 3000–3500 мест;
- нет события `Фестиваль «Васильковый край»` на 3200 мест со схемой;
- максимум 1–2 open-air/capacity-only события;
- open-air/capacity-only events имеют вместимость до 1500;
- события с seatmap имеют вместимость до 500;
- каждое event связано с корректной venue;
- seatmap event имеет venue со схемой;
- capacity-only event имеет venue без схемы;
- `kvitki.by` полностью заменён на `bezkassira` / «Безкассира»;
- регионы Минск, Могилёвская область, Гродненская область имеют полезные демонстрационные данные.

## 2. Regional scope

Итоговое состояние:

- Республиканский уровень видит всё;
- Могилёвский сотрудник видит только Могилёвскую область;
- Гродненский сотрудник видит только Гродненскую область;
- региональные пользователи не видят пустые разделы;
- заявки мероприятий фильтруются по области;
- журнал решений фильтруется по уровню/региону.

## 3. Documents

Итоговое состояние:

- клики по документам открывают визуальный mock-preview;
- preview похож на документ, а не на текстовую карточку;
- стиль: ретро-пиксельная бюрократия / border-control homage;
- watermark `DEMO`;
- данные вымышленные;
- нет реальных персональных данных;
- нет копирования игровых ассетов/скриншотов;
- preview закрывается по кнопке и Escape.

## 4. Compliance tooltips

Итоговое состояние:

- generic tooltip про сохранение черновика отсутствует;

- этап 7 не ломает верстку.
