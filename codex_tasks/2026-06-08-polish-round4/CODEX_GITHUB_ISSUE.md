# Polish round 4: UI/UX demo credibility, fee transparency and non-regression enrichment

## Goal

Поднять демонстрационный уровень `cl_platform` перед срочной презентацией: убрать игрушечность, усилить правдоподобие данных, русификацию, статусы, контроль, админские таблицы, журнал решений, формы и кабинет билетного оператора.

## Why it matters

Проект готовится для показа серьёзным стейкхолдерам. Текущий аудит показал, что основная цепочка уже есть, но впечатление портят технические mock-id, одинаковые постеры, английские UI-строки, слабые статусы, пустой контроль и недостаточно убедительная админская зона.

## Expected outcome

После итерации live demo должно выглядеть не хуже текущего состояния и заметно убедительнее:

- события выглядят как реальная культурная афиша, а не тестовая база;
- типы мероприятий соответствуют площадкам и формату зала;
- постеры разные и вертикальные;
- visible UI на основных маршрутах русский и деловой;
- заявки имеют понятный статусный контекст;
- в Центре Управления есть журнал решений / audit trail;
- раздел `Контроль` содержит осмысленные бизнес-кейсы нарушений;
- админские таблицы читаются лучше;
- формы структурированы по секциям;
- госпошлина показана как mock-элемент demo-flow с понятным расчётом;
- на `/proto` есть переключатель порядка начисления платежей: `По действующим правилам` / `Расширенный расчёт`;
- шаг 8 `Пошлинные платежи` объясняет, почему начислена сумма, включая пример `10 × 42 BYN = 420 BYN`;
- кабинет билетного оператора выглядит как бизнес-интерфейс, а не developer dashboard;
- GitHub Pages обновлены и live smoke-check пройден.

## Out of scope

- не трогать demo-flow входа и авторизацию;
- не менять routes;
- не делать production backend;
- не делать реальные API-интеграции;
- не делать реальный платёжный шлюз;
- не делать реальную ЭЦП;
- не делать новый purchase flow;
- не создавать новую design system;
- не делать глобальный редизайн или ребрендинг;
- не ухудшать текущие рабочие сценарии.

## Delivery requirement

Результат задачи — не PR и не build, а задеплоенные GitHub Pages с успешным live smoke-check.

Codex должен довести задачу до:

```text
implementation → self-check → npm run build → smoke-check → PR → merge → GitHub Pages deploy → live smoke-check → final report
```
